# Sprint 05 Brief — Settings / Admin Master Data

## Goal
Replace hardcoded operational options with practical configurable master data.

## Why this sprint exists
A more mature agency OS should not require code edits for every dropdown/category change.

## Required outcomes
- settings page becomes real
- key master data becomes configurable
- app keeps working without a giant rewrite

## Suggested settings domains
- system roles
- functional roles
- worker types
- disciplines
- project types
- task categories
- deliverable types
- file categories
- rate types
- payment methods
