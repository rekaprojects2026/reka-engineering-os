# Sprint 02 Brief — Invite-Only Onboarding

## Goal
Allow admin to invite team members/freelancers and allow invited users to complete their own profile.

## Why this sprint exists
The owner should not be forced to manually input every field forever.
The system should remain invite-only rather than public signup.

## Required outcomes
- invite flow
- invited user activation flow
- profile completion flow
- clear separation between admin-controlled fields and user-editable fields

## Required behavior
### Admin controls
- system_role
- functional_role
- worker_type
- active status
- approved rate and payment-critical fields

### User can complete
- name
- phone
- city
- portfolio
- skills if supported
- bank/e-wallet info
- availability
- expected rate (optional)

## Important
No public open registration.
No broad self-signup.
