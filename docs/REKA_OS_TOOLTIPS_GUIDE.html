Pembuka — cara memakai dokumen ini

Dokumen ini adalah referensi teks untuk tooltip dan penjelasan modul ReKa Engineering OS (Next.js + Supabase). Gunakan sebagai: (1) salin teks ke atribut title di HTML/React untuk tooltip bawaan browser; (2) salin ke komponen tooltip UI; (3) materi onboarding internal.

Tooltip singkat vs penjelasan penuh. Baris demo dengan teks bergaris putus-putus biru: arahkan mouse atau fokus keyboard (Tab) untuk melihat tooltip hover (teks di data-tip). Di bawahnya, blok <details> berisi penjelasan panjang untuk training / dokumentasi. Untuk produksi React, prefer komponen tooltip yang mengekspos aria-describedby / role="tooltip" pada elemen munculan; di sini pola CSS hanya contoh visual.

Navigasi aplikasi nyata: sidebar AppSidebar menampilkan Dashboard, Operations (Leads, Outreach, Clients, Projects, Tasks saja sesuung role), Finance (Invoices, Compensation, Payslips, Payments, My Payments, plus tautan FX Rates & Payment Accounts ke /settings?tab=finance), People (Team), Admin (Settings). Modul Deliverables, Files, Intakes, Search, dll. diakses dari alur lain (URL langsung, tautan internal, atau menu yang tidak ada di potongan sidebar ini).

Menu di aplikasi nyata tergantung profiles.system_role (8 nilai: direktur, technical_director, finance, manajer, bd, senior, member, freelancer). Jika suatu modul tidak muncul di sidebar Anda, itu sesuai matriks akses — bukan bug.

Ringkasan 8 peran (tooltip inti)

Gunakan baris ini sebagai tooltip singkat di pengaturan tim atau halaman admin:

direktur Persetujuan strategis: approve/reject project, void invoice, payslip final, termin domestik; lihat KPI organisasi; team & settings finance biasanya read.
technical_director Operasional penuh: semua project/task/deliverable/file; buat project (pending direktur); settings sistem & file naming; tidak akses invoice/payslip.
finance Invoice, payslip, pembayaran tim, kompensasi (confirm), FX & rekening (tab Finance di Settings); tidak edit master dropdown sistem.
manajer Project yang dipimpin, task/deliverable di project itu, intake buat/edit; propose kompensasi untuk timnya; tanpa finance penuh.
bd Outreach penuh, client baru & kontak, intake; dashboard sales; tanpa finance & tanpa task operasional.
senior Dashboard personal review: task/deliverable sebagai reviewer, approve versi file terkait.
member Task & deliverable yang di-assign, file pada tasknya; tanpa halaman Projects list.
freelancer Hanya task terisolasi (tanpa nama proyek di dashboard), file & deliverable submit; navigasi minimal.

Konsep silang — Outreach, Client, Intake/Lead, Convert

Outreach vs Client

Outreach Daftar prospek perusahaan atau kontak: status follow-up. Belum jadi master klien. Utama diisi BD. ?
Penjelasan penuh — Outreach

Outreach mencatat siapa yang didekati sebelum ada deal formal. Satu baris = satu entitas prospek (perusahaan/kontak) dengan channel (LinkedIn, email, dll.).

Saat prospek “panas”, gunakan aksi Convert to lead: sistem membuat Intake baru (status new) dengan judul otomatis dari nama perusahaan, dan menandai baris outreach sebagai converted.

Client ?
Penjelasan penuh — Client

Client adalah entitas bisnis resmi di tabel clients. Dipakai untuk menaut proyek, alamat tagihan, dan referensi invoice.

Intake bisa punya temp_client_name (prospek teks) atau client_id (sudah terhubung ke master client).

Dua arti kata “Convert”

  • Convert (Outreach) → membuat Intake/Lead baru dari baris outreach. Biasanya role bd.
  • Convert to Project → membuat Project dari intake yang sudah qualified/closed. Membutuhkan client (UUID atau dropdown di halaman /intakes/[id]/convert) dan project lead.
Outreach (prospek)
  → [Convert to lead] → Intake / Leads (kualifikasi, data scope)
    → link ke Client (opsional tapi disarankan)
      → [Convert to Project] → Project (operasional)
        → Tasks / Deliverables / Files
          → (Domestic) Termin & invoice internal
