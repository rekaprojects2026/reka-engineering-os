# Skill — UI and UX Rules

## Design intent
The application is a serious internal operations tool for an engineering agency.

## Must feel like
- calm
- professional
- efficient
- dense but readable
- desktop-first
- operational

## Must not feel like
- a marketing landing page
- crypto dashboard energy
- overanimated prototype
- oversaturated neon UI
- playful consumer app

## UX rules
- data-heavy pages default to tables
- cards are for summaries only
- quick actions should be easy to find
- use drawers or modals for quick create and quick edit
- keep filters visible
- status colors must be consistent
- empty states should guide the next action
- spacing hierarchy must stay consistent

## Color direction
- background: soft neutral light
- surfaces: white and light gray
- text: dark neutral charcoal
- primary: muted navy / slate blue
- accent: subdued steel blue
- success: muted green
- warning: amber
- danger: brick red or muted red

## Interaction rules
- reduce clicks
- keep headers sticky where useful
- show due dates and statuses prominently
- use icons sparingly and functionally
