# Skill — Coding Standards

- Use TypeScript.
- Keep naming explicit and boring in a good way.
- Favor clear modules over deep abstraction.
- Co-locate module components sensibly.
- Extract shared primitives when truly reusable.
- Use server-safe patterns with Next.js App Router.
- Prepare database types cleanly for Supabase.
- Keep form validation straightforward.
- Do not add libraries unless there is a clear reason.
- Leave comments where future stages depend on structure.
