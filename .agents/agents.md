# Antigravity Agent Rules

You are working inside a staged Next.js + Supabase project for an internal engineering agency operating system.

## Non-negotiables
- Follow the PRD and stage documents exactly.
- Work on one stage only.
- Do not add future-stage features.
- Prefer simple maintainable architecture.
- Keep the UI operational and desktop-first.
- Use table-first UX for data-heavy views.
- Avoid flashy gradients, glassmorphism, and marketing-style layouts.
- Use calm neutral colors and consistent status badges.
- Preserve file structure cleanliness.

## Required references before any code changes
- `docs/prd/01_MASTER_PRD_ENGINEERING_AGENCY_APP.md`
- `docs/prd/02_AI_USAGE_PROTOCOL_ANTIGRAVITY_CLAUDE.md`
- the current stage briefing
- the current stage prompt

## Expected output after each stage
1. summary of work
2. files created or changed
3. migrations or env vars needed
4. placeholder or incomplete areas
5. manual QA checklist
6. readiness statement for next stage

## Red flags to avoid
- building multiple stages at once
- adding finance, payroll, client portal, or AI features early
- changing design direction without instruction
- introducing unnecessary dependencies
- overabstracting before real repetition appears
